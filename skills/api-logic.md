# Skill: API and Business Logic Handling

## When to use
- Creating or modifying API endpoints
- Handling calculations (mileage, fuel cost, etc.)
- Processing or storing data
- Fixing incorrect outputs

## Steps
1. Understand expected input and output
2. Validate all inputs (no null, no invalid values)
3. Review existing logic before modifying
4. Identify minimal change required
5. Ensure calculations are correct and consistent
6. Handle error cases explicitly

## Rules
- Do NOT duplicate logic across files
- Do NOT skip validation
- Do NOT hardcode values unless necessary
- Keep logic centralized and reusable
